STEP 3 — chunk_text(text, chunk_size=800, overlap=100)
Sliding window algorithm — creates overlapping chunks to preserve context at boundaries.
Full text (e.g. 2600 characters)
████████████████████████████████████████████████████████████
Chunk 0
chars 0–799
Chunk 1
chars 700–1499
Chunk 2
chars 1400–2199
Chunk 3
chars 2100–2599
Main chunk (800 chars)
Overlap (100 chars)
Algorithm: start = 0 → while start < len(text) → chunk = text[start : start+chunk_size] → skip if empty → start += (chunk_size − overlap)
🔌 API Endpoints (main.py)
GET /
Homepage — Document List
  • Calls db.list_documents()
  • Returns all docs ordered by upload date DESC
  • Renders index.html with Jinja2
  • Response: 200 HTML
POST /upload
File Upload — Full Ingestion Pipeline
  • Accepts multipart/form-data — list of files
  • For each: UUID → save → DB insert → extract → normalize → chunk → DB insert
  • Unsupported formats silently skipped (ValueError caught)
  • Response: 303 Redirect → /
GET /health
Health Check
  • No logic — immediate response
  • Used by deployment/infra monitoring
  • Response: {"status": "ok"}
GET /documents/{document_id}/chunks
Retrieve Chunks for a Document
  • Calls db.get_chunks_for_document(doc_id)
  • Returns chunks ordered by chunk_index ASC
  • Response: 200 JSON
{
  "document_id": "...",
  "chunk_count": 12,
  "chunks": [
    {"chunk_index": 0, "content": "..."},
    ...
  ]
}

🗄️ Database Schema (SQLite — /data/app.db)
📄 documents
PK doc_id TEXT UUID v4, unique document identifier
filename TEXT Safe name: {uuid}.{ext}
original_name TEXT User's original filename
uploaded_at TEXT ISO 8601 UTC timestamp
1
∞
One document
has many chunks
🧩 chunks
PK id INTEGER Auto-increment
FK document_id TEXT → documents.doc_id
chunk_index INTEGER Sequential position (0-based)
content TEXT 800-char text segment
Database Operations Map
WRITES
init_db() — CREATE TABLE IF NOT EXISTS
insert_document() — INSERT INTO documents
insert_chunks() — INSERT INTO chunks (batch)
READS
list_documents() — SELECT * FROM documents ORDER BY uploaded_at DESC
get_chunks_for_document() — SELECT * FROM chunks WHERE document_id=? ORDER BY chunk_index
⚠️ Each function opens a new SQLite connection and closes it after use. No connection pooling. Uses parameterized queries to prevent SQL injection. Row factory set to sqlite3.Row for dict-like access.
